Filter Candidate Tester

Validate a proposed pre-buy filter BEFORE shipping it. Run any threshold combo against historical trades and see whether it would have helped or hurt.
How it works: Define a "rolling-over momentum" filter by setting 3 thresholds. The tool replays the filter against every trade in the date range and reports:
  • Losses caught: trades the filter would have BLOCKED that actually lost money — these are saves
  • Wins blocked: trades the filter would have BLOCKED that actually made money — these are missed opportunity
  • Net impact: total P&L if the filter had been on, vs the actual P&L without it. Positive = filter helps, negative = filter hurts
Filter logic: block if within_30d_high% ≥ X AND red_days_in_7 ≥ Y AND 30d_return% < Z.

Candidate filter parameters

e.g. 90 = "buy price is within 10% of 30d peak"
how many of last 7 days closed lower than open
only block when 30d return is BELOW this (avoid blocking strong uptrends)
Quick presets: iter47 idea (90% / 4d / <30%) Stricter (95% / 5d / <50%) Looser (85% / 3d / <30%) Off (no blocks)
